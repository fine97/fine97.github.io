    一、文件服务器

    gohttpserver是一个简单易于部署的 HTTP 文件服务。

    详细解释

    • docker run：这是 Docker 命令，用于运行一个新的容器。
    • -d：表示以分离模式（detached mode）运行容器。容器将在后台运行，你不会看到其输出。
    • -p 16666:8000：将宿主机的端口 16666 映射到容器内的端口 8000。这意味着你可以通过访问宿主机的 http://localhost:16666 来访问容器内运行的服务。
    • -v /data/file-server:/app/public：将宿主机目录 /data/file-server 挂载到容器内的 /app/public 目录。这样，容器内的 /app/public 目录会对应宿主机上的 /data/file-server 目录。所有在容器中上传或删除的文件都会反映到宿主机上的该目录。
    • --name gohttpserver：为容器指定一个名字 gohttpserver。你可以通过这个名字来管理该容器。
    • codeskyblue/gohttpserver：这是要运行的 Docker 镜像的名称。codeskyblue/gohttpserver 是一个开源的基于 Go 语言的 HTTP 文件服务器。
    • --upload：这是传递给 gohttpserver 的参数，表示启用文件上传功能。
    • --delete：这是传递给 gohttpserver 的参数，表示启用文件删除功能。
