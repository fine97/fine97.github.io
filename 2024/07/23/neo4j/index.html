    Neo4j 图数据库搭建记录

    发表于 2024-07-23 更新于 2024-07-29 分类于 数据组件 Waline： 本文字数： 595 阅读时长 ≈ 2 分钟

    一、背景

    公司数据团队内部下半年计划有数仓模型进行梳理，治理跨层依赖等场景，需要通过爬虫将数据平台中的依赖关系抓取落地并进行分析。

    针对数仓调度依赖、数据存储等场景，使用图数据库存储和分析有天然的优势。

    在数仓节点/边等有限的背景下，结合 Graph DBMS 排名，使用热度最高的 Neo4j 社区版单机部署足以满足场景需求，且 Neo4j 发展多年，相关文档较为完善，便于遇到线上问题时查阅排除。

    通过 Docker 部署：

    参数详解

    • docker run：启动一个新的 Docker 容器。
    • -d：在后台运行容器，即“detached”模式。
    • --name neo4j：将容器命名为 neo4j。
    • -p 7474:7474：将宿主机的 7474 端口映射到容器的 7474 端口，用于 Neo4j HTTP 接口。
    • -p 7687:7687：将宿主机的 7687 端口映射到容器的 7687 端口，用于 Neo4j Bolt 协议接口。
    • -v /data1/neo4j/data:/data：将宿主机的 /data1/neo4j/data 目录映射到容器的 /data 目录，用于存储数据库文件。
    • -v /data1/neo4j/logs:/logs：将宿主机的 /data1/neo4j/logs 目录映射到容器的 /logs 目录，用于存储日志文件。
    • -v /data1/neo4j/conf:/var/lib/neo4j/conf：将宿主机的 /data1/neo4j/conf 目录映射到容器的 /var/lib/neo4j/conf 目录，用于存储配置文件。
    • -v /data1/neo4j/import:/var/lib/neo4j/import：将宿主机的 /data1/neo4j/import 目录映射到容器的 /var/lib/neo4j/import 目录，用于导入文件。
    • --env NEO4J_AUTH=neo4j/xxxxxx：设置环境变量 NEO4J_AUTH，指定 Neo4j 的认证方式（用户名为neo4j，密码为 xxxxxx）。
    • neo4j:4.4.35-community：指定要运行的 Neo4j 社区版镜像的具体版本 4.4.35。
